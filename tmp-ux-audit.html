Cross-cutting Themes

🎯 Accessibility — Focus States

Most interactive elements lack focus-visible:ring styling. This is the single most impactful a11y fix.

  • CountryPanel — all buttons, pills, clickable chips
  • PillGroup — tab buttons have keyboard nav but no focus ring
  • FilterChip — trigger button needs focus ring
  • CalendarView — row clicks not keyboard-accessible ✅ Fixed
  • DiscoverView — search input, filter pills ✅ Fixed
  • ChatModal — header actions, footer controls
  • SettingsModal — form controls, action buttons
  • ConfirmDialog — action buttons
  • App header — nav buttons, menu items

Fix: Add global .focus-ring utility: focus-visible:ring-2 focus-visible:ring-offset-1 focus-visible:ring-[var(--cp-accent)]

♿ Accessibility — Reduced Motion

Zero prefers-reduced-motion handling. 14+ animations (slideUp, scaleIn, cardAppear, itinerary-card-in, shimmerSweep, etc.) run unconditionally.

  • HoverCard — cardAppear animation
  • ConfirmDialog — slideUp + scaleIn
  • TripsView — shimmerSweep, slideUp for drawer
  • AiItineraryModal — itinerary-card-in, day delay animations
  • ItineraryCinematic — cityPulse, card animations
  • ChatModal — fadeInUp, typing dots

Fix: Add @media (prefers-reduced-motion: reduce) { *, ::before, ::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } } in index.css

👆 Touch Targets

Many controls are below the WCAG 2.2 minimum of 24×24px (target 44×44px). Worst offenders use text-[9px] with py-0.5.

  • PlanCompareModal — comparison chips (text-[9px] py-0.5)
  • ItineraryModal — route pills, nav controls
  • AiItineraryModal — route/save chips, close button
  • ChatModal — header action buttons, Paste/Link/Finish
  • SettingsModal — Show/Delete/Restore buttons
  • DiscoverView — clear filter button ✅ Fixed
  • PillGroup — tab buttons (py-1 is borderline)

Fix: Minimum min-h-[32px] min-w-[32px] on all interactive elements; prefer p-2 over py-0.5 px-1

🎬 Animation Consistency

Mixed timings across the app: 0.15s, 0.2s, 0.25s, 0.35s, 0.4s, 0.5s. No design tokens for motion.

  • 4 unused keyframes: transportOrbit, transportGlow, contrailShimmer, fadeSlideIn
  • Some use transition-all (expensive) vs specific properties
  • No consistent easing — mix of ease-out, ease, and defaults

Fix: Define motion tokens: --motion-fast: 0.15s, --motion-normal: 0.25s, --motion-slow: 0.4s. Remove unused keyframes. Replace transition-all with specific properties.

📱 Mobile UX Gaps

Some portal-based popups and tooltips don't handle viewport edges on small screens.

  • FilterChip — fixed popup position can overflow
  • Tooltip — no mobile fallback, can clip off-screen
  • HomeCountrySelector — dropdown may clip on narrow viewports
  • App header — nav pills + hamburger + selector feel cramped
  • PlanCompareModal — table is very cramped on mobile

Fix: Add viewport-aware positioning to portal popups; full-screen fallback on mobile for complex popups.

⚠️ ARIA Semantics

Several components have incomplete or incorrect ARIA patterns.

  • Tooltip — uses role="img" instead of a button/trigger pattern
  • HomeCountrySelector — missing aria-expanded, aria-controls, listbox semantics
  • FilterChip — missing aria-haspopup, focus management for popover
  • CalendarView — table rows clickable but no keyboard support or aria-selected ✅ Fixed
  • App header — nav buttons lack aria-current for active state
  • PillGroup — has role="tablist" but missing aria-controls, aria-orientation
